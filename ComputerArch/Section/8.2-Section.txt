10 True/False Questions
1.	Virtual memory allows processes to use more memory than physically available.
Answer: True
2.	A page fault always results in a program crash.
Answer: False
3.	The page table maps virtual addresses to disk addresses.
Answer: False (maps to physical addresses)
4.	Memory fragmentation is eliminated in paged virtual memory systems.
Answer: False (external fragmentation is reduced, not eliminated)
5.	The offset field in a virtual address specifies the page number.
Answer: False (specifies location within a page)
6.	Disk storage is slower than main memory, affecting virtual memory performance.
Answer: True
7.	The valid bit in a page table entry indicates whether a page is in physical memory.
Answer: True
8.	Virtual memory requires contiguous allocation of pages in physical memory.
Answer: False
9.	A larger page size always improves system performance.
Answer: False (can lead to internal fragmentation)
10.	Physical addresses are generated by the CPU.
Answer: False (virtual addresses are generated by CPU; OS translates to physical)


Model Answers for Essay Questions
1. Explain the concept of virtual memory and how it enhances system performance.
Answer:
Virtual memory is a memory management technique that allows a system to use disk storage as an extension of main memory (RAM). It enables processes to use more memory than what is physically available by swapping inactive pages between RAM and disk. This enhances performance by:
•	Increasing memory capacity without adding expensive RAM.
•	Allowing multiple processes to run simultaneously by isolating their address spaces.
•	Improving efficiency by loading only required pages into memory, reducing wasted space.
However, excessive paging (thrashing) can degrade performance due to slower disk access.
________________________________________
2. Describe the role of page tables in virtual memory systems.
Answer:
A page table is a data structure used by the operating system to map virtual addresses (generated by a process) to physical addresses (actual RAM locations). Each process has its own page table, which contains entries for every page, storing:
•	Physical frame number (if the page is in RAM).
•	Valid/invalid bit (indicating whether the page is in memory or on disk).
•	Protection bits (read/write/execute permissions).
During address translation, the OS checks the page table to resolve virtual addresses, handling page faults if necessary.
________________________________________
3. What is a page fault, and how does the operating system handle it?
Answer:
A page fault occurs when a process accesses a virtual address whose page is not in physical memory (invalid bit set in the page table). The OS handles it as follows:
1.	Trap to OS: The CPU interrupts the process and transfers control to the OS.
2.	Locate page: The OS checks if the page is on disk (swap space).
3.	Free frame: If memory is full, a victim page is evicted (using algorithms like FIFO or LRU).
4.	Load page: The required page is loaded from disk to a free frame.
5.	Update page table: The valid bit is set, and the process resumes execution.
________________________________________
4. Compare and contrast physical addresses and virtual addresses.
Answer:
Aspect	Physical Address	Virtual Address
Definition	Actual location in RAM.	Logical address generated by a process.
Visibility	Hardware-managed (CPU/MMU).	Process-only (OS translates to physical).
Purpose	Directly accesses memory hardware.	Provides abstraction and isolation.
Size	Limited by RAM capacity.	Can exceed physical memory (via paging).
Fragmentation	Suffers from external fragmentation.	Eliminates external fragmentation.
________________________________________
5. How does memory fragmentation occur in virtual memory systems?
Answer:
•	Internal Fragmentation: Wasted space within a page because a process does not fully utilize the allocated page (e.g., a 3KB process in a 4KB page wastes 1KB).
•	External Fragmentation: In non-paged systems, free memory becomes divided into small, non-contiguous blocks. Paging reduces external fragmentation by allowing non-contiguous allocation, but internal fragmentation remains.
________________________________________
6. Explain the process of address translation in a paged virtual memory system.
Answer:
1.	CPU generates a virtual address (e.g., 0x1553).
2.	Split address: The address is divided into:
o	Page number (high-order bits, e.g., 101 for 0x1553).
o	Offset (low-order bits, e.g., 0101010011).
3.	Consult page table: The OS uses the page number to find the physical frame (e.g., frame 01).
4.	Construct physical address: Combine frame number + offset (e.g., 0x1363).
5.	Access memory: If the page is invalid, a page fault occurs.
________________________________________
7. What are the advantages of using virtual memory over increasing physical memory?
Answer:
•	Cost-effective: Disk storage is cheaper than RAM.
•	Multitasking: Enables more processes to run than physical memory allows.
•	Isolation: Prevents processes from accessing each other’s memory.
•	Flexibility: Supports dynamic memory allocation and large applications.
•	Fragmentation reduction: Paging minimizes external fragmentation.
________________________________________
8. Describe how a page replacement algorithm works when a page fault occurs.
Answer:
When memory is full, the OS must select a victim page to evict. Common algorithms:
•	FIFO: Evicts the oldest page (simple but may remove frequently used pages).
•	LRU (Least Recently Used): Evicts the least recently accessed page (optimal but hard to implement).
•	Clock Algorithm: Approximates LRU using a reference bit (efficient compromise).
The chosen page is written to disk if modified, and the new page is loaded into the freed frame.
________________________________________
9. Why is disk storage used as an extension of main memory in virtual memory systems?
Answer:
•	Capacity: Disks provide large, inexpensive storage compared to RAM.
•	Persistence: Data remains intact even after power loss.
•	Overcommitment: Allows systems to allocate more memory than physically available.
However, disk access is slower (~ms vs. RAM’s ~ns), so excessive paging causes performance degradation (thrashing).
________________________________________
10. How does the size of a page affect system performance in virtual memory?
Answer:
•	Larger pages:
o	Reduce page table size (fewer entries).
o	Lower page fault rate (more data per page).
o	But increase internal fragmentation (unused space).
•	Smaller pages:
o	Minimize internal fragmentation.
o	Increase page table size and fault rate (more frequent disk I/O).
The optimal size balances these trade-offs (typically 4KB–64KB in modern systems).

Solution:
o	Page number = 0x20x2, offset = 0xA30xA3.
o	Physical address = Frame 5 (0x5) + offset (0xA3) = 0x5A3.
4.	A system has 4 page frames. Given the reference string: 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5, how many page faults occur using FIFO replacement?
Solution:
o	Page faults: 1, 2, 3, 4 (4 faults), then 5 replaces 1 (5th fault), etc. Total = 9 faults.
